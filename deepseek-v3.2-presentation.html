The Open-Source Gap

Recent reasoning models (o-series, Gemini) have dramatically widened the gap between proprietary and open-source LLMs.

Three critical deficiencies in open models:

Key Deficiencies
  1. No efficient long-context architecture — standard attention scales $O(L^2)$
  2. No scalable reinforcement learning — RL typically limited to <1% of pre-training compute
  3. No agentic capabilities — poor tool use, multi-step reasoning
2

Three Key Contributions

Architecture

Dynamic Sparse Attention (DSA) — reduces attention from $O(L^2)$ to $O(Lk)$ with hardware-aligned sparsity.

Training

Scalable RL — GRPO consuming >10% of pre-training compute, with innovations for stability at scale.

Capability

Agentic Task Synthesis — 1,800+ environments, 85K+ prompts for tool-use and multi-step agent training.

3

Dynamic Sparse Attention

A two-component system that replaces full attention for long contexts:

Lightning Indexer

Lightweight scoring module that estimates token relevance in $O(L)$:

$$I_{t,s} = \sum_{j=1}^{H^I} w_{t,j}^I \cdot \mathrm{ReLU}\!\bigl(\mathbf{q}_{t,j}^I \cdot \mathbf{k}_{s}^I\bigr)$$

Where $H^I$ attention heads produce importance scores for each source position $s$ relative to target $t$.

4

Fine-Grained Token Selection

The DSA attention output uses only the top-$k$ tokens selected by the indexer:

$$\mathbf{u}_t = \mathrm{Attn}\!\bigl(\mathbf{h}_t,\; \{\mathbf{c}_s \mid I_{t,s} \in \mathrm{Top}\text{-}k(I_{t,:})\}\bigr)$$
Training Strategy

Stage 1: 2.1B tokens — pre-fill training to align indexer with full attention.

Stage 2: 943.7B tokens — decoding training with $k = 1{,}024$ selected tokens per query.

Hardware-aligned: selects contiguous pages of 64 tokens for efficient GPU memory access.

Scaling Reinforcement Learning

Group Relative Policy Optimization (GRPO) at unprecedented scale — over 10% of pre-training compute.

$$J(\theta) = \mathbb{E}_{q \sim \mathcal{D},\, \{o_i\} \sim \pi_{\theta_{\text{old}}}}\!\Bigl[\frac{1}{G}\sum_{i=1}^{G} \min\!\bigl(r_i \hat{A}_i,\;\mathrm{clip}(r_i, 1\!-\!\epsilon, 1\!+\!\epsilon)\,\hat{A}_i\bigr)\Bigr]$$

Four Key Innovations

  • Unbiased KL estimation — avoids systematic underestimation of divergence
  • Off-policy importance weight masking — clips stale samples
  • Keep routing — freezes MoE routing during RL to prevent expert collapse
  • Keep sampling mask — prevents degenerate length exploitation
7

Thinking in Tool-Use

Integrating chain-of-thought reasoning with function calling for multi-step agent tasks.

Context Management

Long tool outputs (>2,000 tokens) are dynamically summarized to prevent context window exhaustion during extended reasoning chains.

Cold-Start Mechanism

RL begins from a supervised baseline — the model first learns tool-use patterns from curated demonstrations before optimization.

Result: seamless interleaving of $\langle\text{think}\rangle$ reasoning blocks with $\langle\text{tool\_call}\rangle$ actions.
