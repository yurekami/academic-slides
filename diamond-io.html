What Is Indistinguishability Obfuscation?

Definition (Indistinguishability Obfuscation)

A PPT algorithm $\mathcal{O}$ is an indistinguishability obfuscator for a circuit class $\{\mathcal{C}_\lambda\}$ if:

Correctness. For all $C \in \mathcal{C}_\lambda$ and all inputs $x$: $\mathcal{O}(C)(x) = C(x)$.

Indistinguishability. For all PPT adversaries $\mathcal{A}$, if $C_0(x) = C_1(x)$ for all $x$, then $|\Pr[\mathcal{A}(\mathcal{O}(C_0))=1] - \Pr[\mathcal{A}(\mathcal{O}(C_1))=1]| \leq \mathrm{negl}(\lambda).$

  • iO is the “best-possible” obfuscation: hides everything except input-output behavior.
  • Implies almost all known cryptographic primitives.

The Bootstrapping Bottleneck

  • Prior constructions (Jain–Lin–Sahai 2021) build iO by bootstrapping from functional encryption (FE).
  • FE-to-iO bootstrapping requires recursively encrypting for each input bit.
  • This creates a tower of nested FE invocations — exponential blowup in depth.
Key Problem

For a circuit on $n$-bit inputs, the FE bootstrap invokes encryption $\Omega(n)$ times recursively, each adding a layer of ciphertext expansion. The result: obfuscation time and output size grow superpolynomially.

II

The Diamond Construction

Key Idea: Matrices Replace Recursion

  • Diamond iO replaces the recursive FE encryption tower with lightweight matrix operations.
  • The circuit $C$ is encoded into structured matrices whose products preserve functionality but hide structure.
Diamond Encoding (Intuition)

Encode $C$ as a product $\mathbf{M} = \mathbf{M}_{\text{top}} \cdot \mathbf{M}_{\text{bot}}$ where $\mathbf{M}_{\text{top}}$ is upper-triangular with entries derived from a pseudorandom oracle (PRO), and $\mathbf{M}_{\text{bot}}$ is lower-triangular. The product masks $C$’s structure while preserving $C(x)$ for any input $x$.

Hardness Assumptions

Assumption 1 (Learning with Errors)

Given $(\mathbf{A}, \mathbf{A}\mathbf{s} + \mathbf{e}) \in \mathbb{Z}_q^{m \times n} \times \mathbb{Z}_q^m$, it is hard to distinguish from $(\mathbf{A}, \mathbf{u})$ with $\mathbf{u} \leftarrow \mathbb{Z}_q^m$ uniform.

Assumption 2 (Evasive LWE)

LWE remains hard even when the adversary can make queries about the secret, provided those queries are evasive — they cannot directly reveal $\mathbf{s}$ or enable its reconstruction via linear combinations.

The All-Product LWE Assumption

Assumption 3 (All-Product LWE — new)

Given $k$ correlated LWE samples $(\mathbf{a}_i, b_i)_{i=1}^{k}$, it is hard to distinguish the product structure across all positions simultaneously from random, even when individual samples are correlated through shared secrets.

  • Introduced in this work; strictly stronger than standard LWE.
  • Captures hardness of distinguishing correlated LWE instances across matrix entries.
  • Essential for proving that the diamond encoding’s matrix products hide circuit structure.

Main Security Theorem

Theorem (Main Result)

Under the Evasive LWE and All-Product LWE assumptions in the pseudorandom oracle model, the diamond construction is a secure indistinguishability obfuscator for all polynomial-size circuits.

Proof strategy.

A hybrid argument transitions from $\mathcal{O}(C_0)$ to $\mathcal{O}(C_1)$ in polynomially many steps. Each hybrid replaces matrix entries using PRO security or reduces to an AP-LWE instance. Evasiveness ensures adversarial queries do not compromise the reductions.

Takeaways

  • Diamond iO provides a direct, non-bootstrapping construction of iO from lattices.
  • Replaces recursive FE encryption with flat matrix algebra — polynomial-time obfuscation.
  • Relies on new All-Product LWE assumption; understanding its hardness is an open problem.
  • Represents a paradigm shift: algebraic masking over cryptographic recursion.
Open Questions

Can AP-LWE be reduced to standard LWE? Can the pseudorandom oracle model be removed? What are the concrete parameters for practical instantiation?
